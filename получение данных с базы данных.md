# 🚀 1. Установление TCP-соединения (Трёхстороннее рукопожатие)

Все взаимодействие между клиентом и сервером происходит по протоколу **HTTP** (или **HTTPS**), который работает поверх **TCP**, обеспечивая надёжную доставку данных.

## 🔄 Установление TCP-соединения
Прежде чем данные начнут передаваться между клиентом и сервером через HTTP (или HTTPS), необходимо установить TCP-соединение:

1. **Клиент отправляет SYN**: Клиент инициирует соединение, отправляя серверу сегмент TCP с флагом `SYN` (synchronize). Это запрос на установление связи.
2. **Сервер отвечает SYN-ACK**: Сервер получает запрос и отвечает сегментом с флагами `SYN` и `ACK` (acknowledgment), подтверждая запрос на соединение.
3. **Клиент отправляет ACK**: Клиент подтверждает ответ сервера, отправляя сегмент с флагом `ACK`. Теперь соединение установлено, и можно начинать передачу данных.

Этот процесс гарантирует, что клиент и сервер синхронизировали свои параметры передачи данных и готовы к обмену информацией.

# 📤 2. Клиент отправляет запрос

После успешного установления TCP-соединения клиент отправляет **HTTP-запрос** (например, запрос на получение данных из базы).

На стороне клиента пользователь выполняет действие, которое инициирует запрос данных. Например, нажатие кнопки или переход на страницу, где должны быть отображены данные. Запрос передаётся как текстовые данные, структурированные по протоколу HTTP.

Пример HTTP-запроса:

```http
GET /books HTTP/1.1  
Host: localhost:3000  
Connection: keep-alive
```
**• GET** указывает, что клиент запрашивает данные с сервера.

**• /books** — это ресурс (например, список книг), который запрашивает клиент.

**• HTTP/1.1** — это версия протокола HTTP.

### 💻 Пример на стороне клиента (React + Axios)

```javascript
axios.get("http://localhost:3000/books")
  .then(response => {
    console.log(response.data)
  })
  .catch(error => {
    console.log(error)
  })
```

# 🌐 3. Запрос отправляется на сервер

Серверное приложение обрабатывает запрос, проверяет его и отправляет запрос к базе данных для получения данных.

```javascript
app.get('/books', (req, res) => {
  db.query('SELECT * FROM books', (err, results) => {
    if (err) {
      return res.status(500).json({ error: 'Database query failed' });
    }
    res.json(results);
  });
});
```

# 📊 4. Сервер взаимодействует с базой данных

Когда сервер получает запрос, он создает SQL-запрос к базе данных для получения нужных данных.

```sql
SELECT * FROM books;
```

# 📦 5. База данных возвращает результат серверу

База данных, получив запрос, выполняет его и возвращает результат серверу. Данные передаются в структурированном виде, обычно как строки или объекты (JSON). Сервер получает эти данные и формирует HTTP-ответ.

Пример данных, возвращаемых базой данных:
```json
[
  { "id": 1, "title": "Book 1", "author": "Author 1" },
  { "id": 2, "title": "Book 2", "author": "Author 2" }
]
```

# 🚚 6. Сервер отправляет данные клиенту

После получения данных из базы сервер формирует ответ (чаще всего в формате JSON) и отправляет его обратно клиенту через HTTP:

Пример HTTP-ответа:
```http
HTTP/1.1 200 OK
Content-Type: application/json

[
  { "id": 1, "title": "Book 1", "author": "Author 1" },
  { "id": 2, "title": "Book 2", "author": "Author 2" }
]
```

**• HTTP/1.1** 200 OK указывает, что запрос был успешным и сервер возвращает данные.

**• Content-Type:** application/json сообщает клиенту, что данные передаются в формате JSON.

Как и в случае запроса, данные HTTP-ответа передаются через TCP в виде сегментов. TCP продолжает обеспечивать надёжность доставки, контролируя порядок пакетов и целостность данных.

1. TCP использует проверку контрольных сумм и повторную передачу потерянных пакетов для предотвращения ошибок.


2. По завершении передачи данных сервер может инициировать закрытие соединения, отправив сегмент с флагом FIN.

# 🖥️ 7. Клиент получает и отображает данные

Когда клиент получает данные от сервера, он использует JavaScript для их обработки и отображения на странице. В приложении на React, например, это может быть использование useState и useEffect для хранения и рендеринга данных:

```javascript
  const [books, setBooks] = useState([]);

  useEffect(() => {
    axios.get('http://localhost:3000/api/books')
      .then(response => {
        setBooks(response.data);
      })
      .catch(error => {
        console.error('Error fetching data', error);
      });
  }, []);
```

# 🔒 Закрытие TCP-соединения
После завершения обмена данными соединение может быть закрыто. Закрытие происходит с помощью процедуры чистого закрытия TCP:

1. Сервер отправляет сегмент с флагом FIN, уведомляя, что данные больше не будут передаваться.

2. Клиент подтверждает получение флага FIN и отправляет свой собственный FIN, после чего сервер подтверждает его, и соединение закрывается.

Этот процесс закрытия соединения гарантирует, что все данные были переданы и ни одно из сообщений не было потеряно.
